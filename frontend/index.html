How it works?

When you start our docker-container with pre-configured Traefik reverse proxy - it allows you to send HTTP requests to other docker-containers, started on your local machine, using domain names and don't care about SSL certificate - our container already contains wildcard *.indocker.app certificate issued by Let's encrypt!

Since this service is located on the .app domain zone, any HTTP requests which are sent from the browser will be automatically rewritten to the HTTPS schema. This occurs because of all domains in this zone are in the HSTS preload list. This affects only browser-based requests - API, curl requests, and so on work as usual.

But what about DNS?

Domain zone *.indocker.app always resolves IP address 127.0.0.1 (100::).

How to start?

Make sure that you have no running nginx or any other application that uses 80 or 443 local ports.

At first, you should create docker network for reverse proxy: 

$ docker network create indocker-app-network
64a84ea622e9a951a38fbd4654c34407ac5b04d516998596ce30d06979391b4e

And then start the container with a reverse proxy in background mode:

$ docker run -d --rm \
  -v /var/run/docker.sock:/var/run/docker.sock \
  -p 80:80 -p 443:443 \
  --network indocker-app-network \
  --name indocker.app \
  tarampampam/indocker-app:1
SSL certificate is valid for just about 3 months after the image download. So, do not forget to update this docker image time-to-time.

How to stop?

Very simple. Just execute in your terminal: 

$ docker stop indocker.app

How can I use this?

Let's write a dead simple docker-compose file that starts nginx with attaching some domain name to it: 

version: '3.8'

services:
  my-nginx:
    image: nginx:latest
    labels:
      - traefik.enable=true
      # https://doc.traefik.io/traefik/routing/providers/docker/#routers
      - traefik.http.routers.my-nginx-router.rule=Host(`my-nginx.indocker.app`)
      - traefik.http.routers.my-nginx-router.service=my-nginx-service
      # https://doc.traefik.io/traefik/routing/providers/docker/#services
      - traefik.http.services.my-nginx-service.loadbalancer.server.port=80
      - traefik.http.services.my-nginx-service.loadbalancer.healthcheck.path=/
      - traefik.http.services.my-nginx-service.loadbalancer.healthcheck.interval=5s
    networks: [indocker-app-network]
    security_opt: [no-new-privileges:true]

  whoami:
    image: containous/whoami:latest
    labels:
      - traefik.enable=true
      - traefik.http.routers.whoami-router.rule=Host(`whoami.indocker.app`)
      - traefik.http.routers.whoami-router.entrypoints=http # force HTTP instead of HTTPS
      - traefik.http.routers.whoami-router.service=whoami-service
      - traefik.http.services.whoami-service.loadbalancer.server.port=8080
    command: --port 8080
    networks: [indocker-app-network]
    security_opt: [no-new-privileges:true]

networks:
  indocker-app-network:
    external: true
$ docker-compose up -d

Now, you can make any requests to the started container using the domain name and HTTPS schema without any additional configuration: 

$ curl -sS https://my-nginx.indocker.app | grep Welcome
<title>Welcome to nginx!</title>
<h1>Welcome to nginx!</h1>
